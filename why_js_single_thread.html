To view the JavaScript output for each experiment, open the browser's developer tools (usually F12 or right-click -> Inspect) and check the "Console" tab.

  • * Javascript is single-threaded language, which means that it can execute only one task at a time. It has a single call stack, which is used to execute the code sequentially.
  • * Javascript is synchronous, which means that it will execute the code line by line, and it will wait for the current line to be executed before moving to the next line.
  • * CallStack is a mechanism how operations are performed in a LIFO (Last In First Out) manner. It is used to keep track of the function calls in the program. Functions invokes new execution context and added to call stack.
  • * Js relys on event loop and callback queue to handle the asynchronous operations. For multti threading js utlizes web workers. to run code parallelly. without blocking the main thread.
  • * Event loop is a mechanism that allows the javascript runtime to perform non-blocking I/O operations. It is responsible for handling the asynchronous operations in javascript.
  • * Event loops get trigger on invocation of the callback function, User interaction, network request, timer, etc. It keeps checking id call stack is empty it looks for event queue and push the callback function to call stack.
  • * Event Loops allow js to perform task without freezing the entire application. It is responsible for handling the asynchronous operations in javascript.
  • * Javascript is loosely typed language, which means that you don't have to specify the data type of the variable when declaring it
  • * The data type of the variable is determined at runtime based on the value assigned to it. (Less Strict)
